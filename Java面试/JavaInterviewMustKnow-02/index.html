(*):重点掌握

Java 基础 （二）

(*)一、面向对象三大特征

面向对象的编程:“万物皆对象”;以对象为中心，以消息为驱动，程序=对象+消息;

继承

访问权限(通过向类或类的成员添加访问修饰符，控制可访问的范围)

修饰符 访问级别 同类 同包 子类 不同包
public 公开 √ √ √ √
protected 受保护 √ √ √
默认 √ √
private 私有 √

继承:子类在继承父类属性和行为的同时，也具备自己特有属性或行为。提高代码的复用性。

super 关键字

  • 访问父类的构造函数:可以使用 super() 函数访问父类的构造函数，从而委托父类完成一些初始化的工作。
  • 访问父类的成员:子类重写父类的某个方法，可以使用 super 调用父类的方法实现。

封装

封装:将抽象出的数据和操作包围，对数据的访问，只能通过特定的方式(get、set)；属性私有化，操作公开化；提高了数据隐秘性和代码的复用性。

抽象

抽象:将现实世界中的某一类事务进行分析，将特征提取出来，用程序代码表示；数据抽象为类属性，行为抽象为类方法。

多态

多态:同一父类的不同子类对象，对同一消息做出的不同响应；动态链接，父类引用(或接口)可以指向子类对象(或实现类)；接触了子类与父类之间的耦合度。

子类 is a 父类

实现类 has a 接口

重写和重载

  • 重写(Override):存在于继承体系中，子类重写父类的方法。
  • 重载(Overload):存在于同一个类中，覆盖类中的某一个方法。

区别:

  1. 重写存在于继承体系，重载存在于同类。
  2. 方法名相同；重写是完全相同的方法，访问权限:子 >= 父，返回值类型 父 >= 子类，异常类型 父 >= 子类；
    重载是参数列表(类型、个数、顺序)至少有一个不同。

  3. 重写需要满足里氏替换原则

    子类访问权限 >= 父类访问权限

    子类返回值类型 <= 父类返回值类型

    子类异常类型 <= 父类异常类型

重载不用。

返回值不同，不算是重载

(*)二、抽象类、抽口

抽象类

抽象方法:使用abstract关键字修饰的方法，不能有实现；

abstract void func();

抽象类:可以包含抽象方法的类，就是抽象类；抽象类不能被实例化，必须通过实现抽象方法的子类实例化。

接口

接口是抽象的延伸。

  • Java 8 之前，接口是完全抽象的类，所有方法都是抽象的。
  • Java 8 开始，接口可以有默认方法的实现。
  1. 接口成员默认是public，不允许使用protected、private。
  2. 成员变量默认为 public static final,也就是常量。
在 Java 8 之前，如果一个接口想要添加新的方法，那么要修改所有实现了该接口的类。

接口与抽象类的区别

  • 设计上:抽象类是一种is a关系，必须满足里氏替换原则(即子类对象必须能替换掉所有父类对象)；
    接口是一种has a关系，提供一种方法实现的协议。
  • 使用上:一个类可以实现多个接口，但只能继承一个抽象类。
  • 编写上:接口的成员默认是public，不能使用protected、private；抽象类没有限制。
  • 属性上：接口的成员变量必须是静态常量；抽象类没有限制。

(*)三、Object 方法

equals()

equals()具有四个特性

  1. 自反性

对于非空引用x
x.equals(x) == true

  1. 对称性

x.equals(y) == y.equals(x)

  1. 传递性

  1. 一致性
    多次调用x.equals(x)结果相同

  2. 与 NULL 相比

对于非空引用x
x.equals(null) == false

equals() 与 ==

  • 基本类型:==判断两个值是否相等，基本类型没有equals()方法
  • 引用类型:==判断两个变量是否引用同一个对象，equals()判断引用的对象是否相等

重写 equals()

  1. 检查是否为同一个对象的引用
  2. 检查是否是同一个类型
  3. 将 Object 对象进行转型，判断每个关键域是否相等

hashCode()

计算散列值

  • 等价的两个对象(equals()为true)散列值一定相同
  • 反之，则不成立
  • equals()比较了哪些域，hashCode()就会对哪些域进行hash值的处理；如:Object 比较和散列的数据是对象的内存地址空间，String 比较和散列的数据是字符串所在的堆空间，基本数据类型比较和散列的数据是其数值。
重载equals()时，hashCode()也要重载,保证等价的两个对象散列值相同

clone()

一个类如果使用clone()必须实现 Cloneable 接口，但clone()是 Object 的方法，实现 Cloneable 接口只是规定。

不实现 Cloneable 接口又调用了clone()，抛出 CloneNotSupportedException 异常。

浅拷贝和深拷贝

  • 浅拷贝:拷贝对象和原始对象的引用类型引用同一个对象
  • 深拷贝:拷贝对象和原始对象的引用类型引用不同的对象

clone() 替代方法

可以使用静态拷贝工厂或拷贝构造器替代clone()

toString()

返回表示对象值的字符串

(*)四、关键字

final

声明变量为常量

  • 基本数据类型: 数值不变
  • 引用数据类型: 引用对象不能变，但对象本身可以修改

声明方法不能被子类覆盖

private 方法隐式地被指定为 final，如果在子类中定义的方法和基类中的一个 private 方法名相同，此时子类的方法不是重写基类方法，而是在子类中定义了一个新的方法。

声明类不允许被继承

static

修饰变量为静态变量

  • 静态变量:类变量，属于类，类的所有实例共享该变量，通过类名访问。
  • 静态常量:静态变量加上 final 关键字。
  • 实例变量:属于类的实例，类的实例对象私有变量，通过对象名访问。

修饰方法为静态方法

  1. 静态方法在类加载之前就已存在。
  2. 静态方法必须有实现，也就是不能是抽象方法。
  3. 静态方法只能访问所属类的静态字段和静态方法。
  4. 静态方法中不能有 this 和 super 关键字，因为这俩个关键字与实例对象相关

静态语句块

在类初始化时运行一次

静态内部类

  1. 非静态内部类实例化依赖于外部类的实例，而静态内部类不需要。
  2. 静态内部类不能访问外部类的非静态的变量和方法。

初始化顺序

  • 静态>普通>构造
  • 静态变量和静态语句块的初始化顺序取决于它们在代码中的顺序

存在继承的情况下，初始化顺序为：

  1. 父类（静态变量、静态语句块）
  2. 子类（静态变量、静态语句块）
  3. 父类（实例变量、普通语句块）
  4. 父类（构造函数）
  5. 子类（实例变量、普通语句块）
  6. 子类（构造函数）
