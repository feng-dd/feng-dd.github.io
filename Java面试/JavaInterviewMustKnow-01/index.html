自动装箱和拆箱(基本数据类型与对应包装类之间的转换)

缓存池

  • 当创建对象时，先判断值是否存在于缓存池中。
  • 若存在，直接使用缓冲池中的对象(即返回缓存池的对象)。
  • 若不存在，创建新的对象。

基本类型对应的缓冲池

  • boolean: true & false
  • byte : all
  • short & int : -128 ~ 127
  • char: \u0000 to \u007F

造成这种情况的原因是：

  1. new Integer(120); 每次都会创建一个新对象。
  2. Integer.valueOf(120); 会使用缓存池中的对象，多次调用的是同一个对象。
  3. Integer m = 120;自动装箱时，编译器会调用valueOf()方法,因此，多次使用自动装箱创建，引用的是同一个对象。

(*)二、String

String被声明为final，因此不能被继承

Java 8 中，String底层利用 char[] 存储数据。

Java 9 中，String底层利用 byte[] 存储数据,coder 识别编码。

value 数组被 final 修饰，因此在初始化之后，value 不能再引用其他数组。
String 内部也没有改变 value 数组的方法，保证了 String 的不可变性。

String 设计成不可变的原因

  1. 缓存 hash 值

String 的不可变性保证了 hashcode 的唯一，利用 String 中的 hash 属性对 String 对象的 hashcode 缓存，
避免了重新计算 hashcode。也是 Java 中 HashMa p等容器将 String 类型作为键的原因。

  1. 常量池

如果一个 String 对象已经创建过，就会指向 String Pool 中已创建的对象。String 的不可变性，
保证了 String 对象可以入池。

  1. 线程安全(同步)

String 的不可变性保证了多线程可以同时操作同一个 String 对象，不用考虑线程安全问题使用同步。

  1. 安全性

数据库的用户名和密码是以字符串的形式传输来获取数据库连接；
socket 编程中，主机名和端口以字符串形式传输；String 的不可变性，防止外部恶意篡改数据，保证了安全性。

String、StringBuffer、StringBuilder

类型 对象可变性 线程安全性
String 不可变 安全
StringBuffer 可变 安全(Synchronized)
StringBuilder 可变 不安全

字符串常量池(String Pool)

字符串常量池保存着所有字符串的字面量，字面量在编译器就已经确定。

String 的 intern() 方法

当字符串调用 intern() 方法时，若常量池中存在与该字符串值相同(使用 equals() 方法判断)的字符串，
返回该字符串的引用；否则在常量池中添加这个新的字符串，并返回该字符串引用。

  • Java 7 之前，String Pool 放在运行时常量池中，属于永久代。
  • Java 7 中，String Pool 放在堆内存中。因为永久代空间有限，大量使用字符串，会造成 OutOfMemoryError 错误。

String 构造函数之一:将一个字符串作为另一个字符串的构造参数时，俩个字符串指向同一个 value 数组

三、参数传递

在 Java 中的，所有参数都是以值传递的方式传入方法中。

参数是基本数据类型时，传递的是参数值的拷贝

输出结果是：
100
500
100

原理：传入test()方法的，是 num 属性值的拷贝，对形参的改变并不会使实参有变化。

参数是引用数据类型时，传递的是引用对象的内存空间地址；当方法内改变该对象的属性值时，就是永久改变。

参数是 String 类型时，传递的是内存空间地址；当方法内对字符串改变时，由于 String 类型的不可变性，
会创建一个新的 String 对象，而方法外的 String 引用的不是新的对象，所以方法外 String 值没变。

四、类型转换

基本数据类型之间的转换

  1. 布尔型不可以转换
  2. byte 与 char 不可以互相转换(取值范围决定)
  3. short 与 char 不可以互相转换(取值范围决定)
  4. byte 可以自动提升到 short(取值范围决定)

强制类型转换

  1. 取值必须在转换类型的范围内,否则会发生数据错误

  2. double 转 float 可能出现精度丢失

进行运算时，数据类型会自动提升至运算符俩侧的较大类型，依次类推；运算完毕后，赋值符号左侧的类型较小时，需要强转。
