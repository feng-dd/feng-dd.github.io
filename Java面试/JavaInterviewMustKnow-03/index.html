Java 基础 （三）

一、反射

反射是什么？

  • “正射”：使用的类是已经确定好的，直接对该类实例化使用。
  • 反射：在运行状态时，才确定要使用哪个类，此时无法使用 new关键字创建对象。通过动态获取对象信息和调用对象方法的功能。

反射获取 Class 对象

反射就是获取类的字节码文件(.class),通过 Class 对象获取类的信息。

获取 Class 对象有三种方式：

通过反射初始化类的对象

获取类对象有俩种方式

通过反射获取类的信息

使用 java.lang.reflect 类库

  • Field 类：可以使用get()和set()方法读取和修改Field对象关联的字段
  • Method 类：可以使用invoke()方法调用与Method对象关联的方法
  • Constructor 类：可以用Constructor创建新的对象。

反射的优缺点

反射的优点：

  1. 可扩展性:运行时动态加载。
  2. 获得类目录:获取类的属性和方法(无法获得方法的实现)。
  3. 调试或测试:利用反射调用类中的方法。

反射的缺点：

  1. 性能开销:反射涉及了动态类型的解析，所以 JVM 无法对这些代码进行优化。
  2. 安全限制:使用反射技术要求程序必须在一个没有安全限制的环境中运行。
  3. 内部暴露:由于反射允许代码执行一些在正常情况下不被允许的操作（比如访问私有的属性和方法），所以使用反射可能导致代码功能失调并破坏可移植性。

二、异常

Throwable 是 Java 中的异常类,分为 Error 和 Exception。

  • Error：JVM 无法处理的错误
  • Exception：分为RuntimeException(非检查异常)和CheckedException(受检查异常)
    1. 受检异常 ：需要用try...catch...语句捕获并进行处理，并且可以从异常中恢复；
    2. 非受检异常 ：是程序运行时错误，例如除 0 会引发Arithmetic Exception，此时程序崩溃并且无法恢复。

JVM 处理异常流程

一般来说，发生异常时，堆上会生成一个异常对象「包含当前栈帧的快照」；
然后停止当前的执行流程，将异常对象从当前 Context 抛出；
然后交由异常处理机制处理异常，使程序继续执行。

异常的处理方式

遇到问题不进行具体处理，而是继续抛给调用者(throw,throws)

try…catch…`捕获异常，针对处理

三、泛型

Java 5 引入的新特性，允许编译时类型安全的检测。

常用于容器、集合对其内部元素的约束

泛型方法

该方法调用时可以接受不同类型的参数，根据传递给泛型方法的参数类型，编译器适当处理每一个方法的调用。

1. <? extends T>表示该通配符所代表的类型是 T 类型的子类。 
2. <? super T>表示该通配符所代表的类型是 T 类型的父类。

类型擦除

使用泛型的时候加上的类型参数，会被编译器在编译的时候去掉。这个 过程就称为类型擦除。

四、注解

注解是一种特殊的注释，需要解析；所有的注解类都继承 Annatation 接口。

注解的本质：Annatation 接口的子接口。

如：JDK 内置注解

  1. @Override：作用于方法之上,编译结束后将被丢弃，必须继承父类的非私有方法。

反编译后

  1. @Deprecated：作用目标任意，永久存在，标记当前的类或方法已过时。
  2. @SuppressWarnings：作用目标任意，编译结束后将被丢弃，消除 java 警告。

解析类或方法的注解方式有俩种

编译期直接扫描：编译器在代码编译字节码的过程中会检测到某个类或者方法被一些注解修饰，这时它就会对注解进行某些处理。

  • 对于 JDK 内置的注解，编译器会按照注解对类或方法进行处理。
  • 对于自定义注解，编译器不知如何对类或方法进行处理，只会根据该注解的作用范围选择是否编译进字节码文件

运行期反射：永久存活的注解，可以通过反射获取注解

  • getAnnotation：返回指定的注解
  • getAnnotations：返回所有的注解
  • getDeclaredAnnotation：返回本元素的指定注解
  • getDeclaredAnnotations：返回本元素的所有注解，不包含父类继承而来的
  • isAnnotationPresent：判定当前元素是否被指定注解修饰

元注解：用于修饰注解的注解

  1. @Target：注解的作用目标,有一个 value 属性 @Target(value = ElementType.FIELD)
    1. ElementType.TYPE：允许被修饰的注解作用在类、接口和枚举上
    2. ElementType.FIELD：允许作用在属性字段上
    3. ElementType.METHOD：允许作用在方法上
    4. ElementType.PARAMETER：允许作用在方法参数上
    5. ElementType.CONSTRUCTOR：允许作用在构造器上
    6. ElementType.LOCAL_VARIABLE：允许作用在本地局部变量上
    7. ElementType.ANNOTATION_TYPE：允许作用在注解上
    8. ElementType.PACKAGE：允许作用在包上
  2. @Retention：注解的生命周期,有一个 value 属性 @Retention(value = RetentionPolicy.RUNTIME)
    1. RetentionPolicy.SOURCE：当前注解编译期可见，不会写入 class 文件
    2. RetentionPolicy.CLASS：类加载阶段丢弃(不能进入方法区)，会写入 class 文件
    3. RetentionPolicy.RUNTIME：永久保存，可以反射获取
  3. @Documented：注解是否应当被包含在 JavaDoc 文档中
  4. @Inherited：是否允许子类继承该注解

五、Java 8 新特性

1.函数式接口：只有一个抽象方法的接口，这个抽象方法定义了这个接口的目标及用途,可选择是否使用 @FunctionalInterface 注解标识函数式接口。
当时用该注解时，该函数式接口不能定义（非静态和非默认）方法，可以定义 Object 类的所有方法，静态方法和默认方法。

2.Lambda 表达式：允许把函数作为一个方法的参数（函数作为参数传递进方法中），
Lambda表达式最终返回的是函数式接口的一个对象实例，因此需要依赖于函数式接口。
相当于替换了匿名类使用。

Lambda的四个特性：

  1. 可选类型声明：传入的参数可以不用声明类型，编译器可以自动识别参数类型。
  2. 可选的参数圆括号：只有一个传入参数时可以省略圆括号(必须同时省略参数类型)，多个参数时必须使用圆括号。
  3. 可选的大括号：如果主体只有一个语句，可以不需要使用大括号。
  4. 可选的返回关键字”return”：如果需要返回值主体只有一个表达式则可以同时省略大括号和”return”关键字，编译器会自动将结果作为返回值，如果使用了大括号则返回值不能省略”return”关键字。

3.Stream：Java 8 中的 Stream 是对集合（Collection）对象功能的增强，java.util.stream.Stream

将要处理的元素集合看作一种流， 流在管道中传输， 并且可以在管道的节点上进行处理， 比如筛选， 排序，聚合等。元素流在管道中经过中间操作（intermediate operation）的处理，最后由最终操作(terminal operation)得到前面处理的结果。

集合有两种方式生成流：

  • stream() − 为集合创建串行流。
  • parallelStream() − 为集合创建并行流。

并发模式（parallelStream）能够充分利用多核处理器的优势，使用 fork/join 并行方式来拆分任务和加速处理过程

4.默认方法：接口可以定义默认方法(default)，对接口进行了扩展，无需因接口添加新方法，导致所有实现类都需进行改动。

5.Optional：使用Optional来包装方法的返回值，这就表示方法的返回值可能为null，
需要使用Optional提供的方法来检查，如果为null，还可以提供一个默认值

6.新的时间和日期API:新的时间和日期API都是不可变的，并且是线程安全的。
LocalDateTime, LocalDate, LocalTime, Duration, Period, Instant, DateTimeFormatter等等API。
